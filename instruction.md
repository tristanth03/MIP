# Report Hub

A static GitHub Pages site that serves as a searchable archive for standardized HTML reports.

## What it does

- **`index.html`** — The hub page. Loads `reports.json`, displays a filterable/searchable/sortable list in a side panel, and renders full reports in an iframe on the right. Supports download and open-in-new-tab. Uses the same navy/gunmetal theme toggle as the reports themselves.
- **`reports/`** — Folder containing all HTML report files. Each report is a self-contained, standardized HTML document with embedded metadata (title in `<title>`, category in `.rl`, date in the footer `.rf`).
- **`reports.json`** — A flat JSON manifest of all reports (title, category, date, filename). Auto-generated by the build script. Loaded client-side for search/filter — no backend needed.
- **`build_manifest.py`** — Scans `reports/`, parses each HTML file for metadata, writes `reports.json`. Run after adding/removing reports.
- **`add_report.py`** — Convenience wrapper: copies one or more HTML files into `reports/`, then runs `build_manifest.py`.

## Adding reports

```bash
# Option A: copy manually, then rebuild
cp /path/to/new_report.html reports/
python3 build_manifest.py

# Option B: one step
python3 add_report.py /path/to/new_report.html
```

Then commit and push. GitHub Pages handles the rest.

## Report metadata format

The build script extracts three fields from each HTML file:

| Field    | Source                                                                 |
|----------|------------------------------------------------------------------------|
| Title    | `<title>` tag                                                          |
| Category | First element with `class="rl"` (e.g. `Mathematics Report`)           |
| Date     | Second `<div>` inside `<footer class="rf">`, format `Updated: (HH:MM) DD.MM.YYYY` |

## Theme

The hub uses the identical CSS variable system as the reports (navy / gunmetal toggle), so everything looks cohesive. Theme preference persists via `localStorage`.
